# CI/CD

YuyuCode's CI pipeline runs entirely on GitHub Actions. There is no local build step — the Vite build and APK signing happen in the cloud on every push.

## Workflows

### `build-apk.yml` — APK Build

Triggers on every push to `main`.

```
1. Install Node deps (cached)
2. npm run build → dist/
3. Java 21 + Android SDK 34
4. cap sync android
5. Restore icons (CI resets them)
6. Auto-bump versionCode = GitHub run number
7. ./gradlew assembleRelease
8. Sign APK with keystore
9. Upload artifact
10. Create GitHub Release (only if commit starts with "release:")
```

Commits that only touch `.md` files skip CI entirely.

### `quality.yml` — Quality Gate

Runs ESLint + vitest on every push:

```bash
npm run lint    # must be 0 problems
npx vitest run  # must be 1235/1235 pass
```

### `codeql.yml` — Security Scan

GitHub CodeQL SAST analysis. Runs on push and weekly schedule.

### `bench.yml` — Benchmark

Runs `yuyu-bench-ci.cjs` on Ubuntu x86_64, stores results in `gh-pages` branch at `dev/bench/`, and deploys the custom benchmark dashboard. Alerts via commit comment if any benchmark regresses >150% from baseline.

### `docs.yml` — Documentation

Builds VitePress and deploys to GitHub Pages. Triggers only when files under `docs/` change.

## Releasing

```bash
node yugit.cjs "release: v4.3 — description of changes"
```

`yugit.cjs` detects the `release:` prefix and:
1. Bumps `version` in `package.json` to `v4.3`
2. Commits + pushes
3. CI triggers `build-apk.yml`
4. A GitHub Release is created with the signed APK attached

The release commit message format must be exact: `release: vX.Y` or `release: vX.Y.Z`.

## Secrets

All secrets are configured in **Settings → Secrets and variables → Actions**:

| Secret | Used by |
|--------|---------|
| `VITE_CEREBRAS_API_KEY` | Build — baked into APK |
| `VITE_GROQ_API_KEY` | Build — baked into APK |
| `VITE_TAVILY_API_KEY` | Build — baked into APK |
| `ANDROID_KEYSTORE` | APK signing — base64 encoded |
| `KEYSTORE_PASSWORD` | APK signing |
| `KEY_ALIAS` | APK signing |
| `KEY_PASSWORD` | APK signing |
| `SONAR_TOKEN` | SonarCloud analysis |
| `DEEPSOURCE_DSN` | DeepSource analysis |

### Keystore Encoding

```bash
openssl base64 < my.keystore | tr -d '\n'
```

Not `base64 -w 0` — the output format differs and CI will fail silently.

## Critical Constraints

::: danger Never remove
```json
"overrides": { "rollup": "npm:@rollup/wasm-node" }
```
This override in `package.json` makes Vite build on Termux ARM64. Without it, `npm run build` fails with a native binary error. CI uses Ubuntu x86_64 and doesn't strictly need it, but removing it breaks local builds entirely.
:::

::: danger Never upgrade
`vitest@3` is pinned. `vitest@4` crashes with `Illegal instruction` on Snapdragon 680 (ARM64). The crash is in the test runner's native bindings — not recoverable.
:::

::: danger Never edit manually
The `android/` folder is generated by Capacitor (`cap sync android`). Manual edits will be overwritten on the next sync. Any Android-level customisation must go through Capacitor plugins or `capacitor.config.ts`.
:::
