MASTER PLAN — WORLD KILLER ROADMAP
═══════════════════════════════════════════════════════════

Penelitian mendalam sudah dilakukan. Ini roadmap komprehensif berdasarkan
gap nyata vs Spck, VS Code, Zed, Cursor, dan semua editor yang ada.

─────────────────────────────────────────
FASE 1 — Editor Core (ngalahin Spck dulu)
─────────────────────────────────────────

[ ] 1. MULTI-TAB EDITOR (PRIORITAS TINGGI)
    - useFileStore sekarang: selectedFile (satu file)
    - Target: openTabs[] array, activeTabIdx, tiap tab punya path+content+dirty+CM view instance
    - UI: tab bar di atas editor, close button per tab, unsaved dot indicator
    - CodeMirror: tiap tab punya EditorState tersendiri, swap state saat ganti tab
    - Effort: medium-heavy

[ ] 2. TYPESCRIPT/JS LANGUAGE SERVER (KILLER FEATURE)
    - @valtown/codemirror-ts atau @marimo-team/codemirror-languageserver
    - TypeScript server di Web Worker via comlink — tidak block UI
    - Hasil: autocomplete yang tahu return type useState, hover type info, inline error
    - Package yang dibutuhkan: @codemirror/autocomplete, comlink, typescript
    - Effort: heavy, payoff terbesar

[ ] 3. INLINE AI GHOST TEXT (COPILOT-STYLE)
    - codemirror-extension-inline-suggestion
    - fetchFn async ke Cerebras/Groq API — ghost text yang bisa Tab untuk accept
    - Ini yang Spck tidak punya sama sekali
    - Effort: medium

[ ] 4. VIM KEYBINDINGS
    - @replit/codemirror-vim
    - vim() extension sebelum basicSetup
    - Toggle via /vim slash command atau config panel
    - Effort: ringan, satu hari

[ ] 5. EMMET ABBREVIATION
    - @emmetio/codemirror6-plugin (disponsori CodePen + Replit)
    - div.container>ul>li*3 → full HTML via Tab
    - Aktif untuk HTML/JSX/TSX
    - Effort: ringan

─────────────────────────────────────────
FASE 2 — Visual Power Features
─────────────────────────────────────────

[ ] 6. EXTRA KEYBOARD ROW (MOBILE KILLER)
    - Row custom di atas soft keyboard: {} [] () ; => // Tab :
    - Pure React div, position sticky, detect keyboard via visualViewport resize
    - Tidak butuh native Android keyboard customization
    - Ini yang Spck punya dan YuyuCode belum ada
    - Effort: medium

[ ] 7. LIVE HTML/CSS/JS PREVIEW
    - iframe dengan srcdoc, debounced update 300ms
    - Combine HTML+CSS+JS jadi satu dokumen
    - Intercept console.log via postMessage → tampil di panel bawah
    - Split: editor kiri, preview iframe kanan
    - Effort: medium

[ ] 8. MINIMAP
    - @replit/codemirror-minimap
    - Mode 'blocks' lebih cocok untuk mobile (lebih ringan dari 'characters')
    - Berfungsi sebagai scroll thumb visual
    - Effort: ringan

[ ] 9. INLINE LINT MARKERS
    - @codemirror/lint
    - Jalankan ESLint/syntax check via Termux server, inject hasilnya sebagai gutter markers
    - Red squiggly lines persis VS Code
    - Effort: medium

[ ] 14. COLLABORATION REALTIME

    - Effort: ringan

GOTCHAS PENTING YANG SUDAH DIPELAJARI
═══════════════════════════════════════════════════════════

1. ESLint react-hooks/refs
   - termRef.current di dalam array literal yang di-render = warning
   - Fix: extract ke komponen terpisah. Component boundary memutus ESLint trace.
   - Jangan pakai eslint-disable-next-line react-hooks/refs — tidak akan bekerja
     karena rule trace melalui function calls, bukan hanya next line

2. xterm.js layout di WebView
   - JANGAN import '@xterm/xterm/css/xterm.css' global — corrupt flex layout
   - Inject CSS minimal via <style> tag dalam komponen
   - Container xterm perlu: minHeight:0, position:'relative', overflow:'hidden'
   - FitAddon.fit() harus dipanggil dalam requestAnimationFrame setelah mount

3. CodeMirror import
   - 'codemirror' (bukan '@codemirror/basic-setup' — deprecated)
   - Compartment untuk dynamic reconfigure tanpa destroy/remount

4. String escape di JSX/Python
   - '\|' di JSX string = invalid escape → pakai '[|]' dalam regex atau '\\\\|'
   - Python heredoc dengan backslash sering corrupt JSX string constants
   - Selalu verify dengan grep setelah Python string replacement

5. vitest@1 wajib
   - vitest@4 crash silent di Termux ARM64
   - Jangan upgrade vitest

6. generateDiff format baru
   - Output: "- L3: teks" / "+ L3: teks" (dengan line number)
   - Tests expect format ini — jangan ubah tanpa update tests

7. CI paths-ignore
   - Push yang cuma ubah .md files → skip CI otomatis
   - workflow_dispatch tetap bisa trigger manual

8. Panels split
   - panels.jsx sekarang barrel re-export
   - Edit panel: buka file yang sesuai (panels.git.jsx, panels.agent.jsx, dll)
   - Jangan edit panels.jsx langsung — isinya cuma export statements
